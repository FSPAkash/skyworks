7Observations & Considerations

Data Volume

Total database size across the 18 in-scope databases is approximately 7.2 TB. Three databases, Skyworks_ODS (~2.0 TB), LIB_ECC_RTP (~2.6 TB), and Skyworks_WH (~1.4 TB), account for approximately 86% of total volume.

Flat File Dependencies

111 SSIS job steps reference flat file data sources via a local server path (E:\Project\FlatFileDataSources\). These represent upstream data inputs outside the SQL Server catalog and will need to be inventoried during the lineage mapping phase to ensure end-to-end flow coverage.

SSIS Package Reuse Pattern

Several packages follow a parameterized reuse pattern: the same .dtsx file is invoked by multiple agent jobs with different runtime parameters (for example, plant code, environment, or date range). This is a significant design pattern that will shape how lineage is mapped, as a single package node in the lineage graph may represent multiple logical data flows.

Legacy Job Footprint

68% of agent jobs are currently disabled. While these are not active data flows, they represent historical lineage context. Several jobs are explicitly flagged for deletion in their naming convention.

9Further Details Requested

To progress the lineage mapping, the following inputs are requested from the Skyworks team.

  1. Object-level query logs across the ODS. Required to identify actively consumed datasets and prioritize them accordingly.
  2. Report metadata or report definition files. Required to assess object usage across the reporting layer for building the data lineage.
  3. A consolidated job-to-package-to-flat-file mapping. We reviewed the SSIS packages residing in the SSISDeployed network folder and attempted to map them to their corresponding SQL Agent jobs by examining the Step Command entries. While a number of packages were successfully identified within the folder, several could not be located in SSISDeployed. Please confirm whether any documentation exists on your end that enumerates all active or currently utilized SQL Agent jobs along with their associated SSIS packages and flat files, mapped against one another.
