03 / 14
Current State

One Data Store, two jobs, and the cost shows up at peak.

The ODS carries both transactional processing and analytical reporting on the same footprint. That overlap is where performance ceilings, governance blind spots, and compute inefficiency originate.

ODS size
1.7TB
Compute
22CPUs
Active SSIS jobs
~250
Peak concurrency
1,231batch/s
Performance
Transactional and analytical workloads contend for the same resources, constraining throughput at peak batch concurrency.
Governance
Limited lineage and metadata visibility makes ownership, dependencies, and downstream impact hard to see.
Cost
A single tier doing both jobs leaves compute spend poorly matched to the value each dataset actually delivers.

The medallion target: Bronze (raw), Silver (conformed), and Gold (curated) separate these concerns. This engagement decides what moves where, in what order, and at what impact.

C · Collection

Assemble the raw material: metadata, logs, pipelines.

Connect to the ODS and capture everything the later layers depend on. Read-only, automated where possible, and surfaced as a clean inventory.

Activities · weeks 1-3
  • 01Connectivity validation: confirm ODS access and identify any constraints.
  • 02Automated metadata ingestion: tables, schemas, columns, sizes.
  • 03Query-log analysis: usage patterns, frequency, access tools.
  • 04SSIS package parsing: map transformation logic flowing through the ODS.
  • 05BI / reporting metadata capture for reports drawn from the ODS, where available.
  • 06Data ownership identification + lineage reference system availability check.

Boundaries

What this engagement does not do.

Clear edges keep the assessment fast and low-risk. The items below are explicitly excluded, and handled by Skyworks or a designated partner, or scoped separately via change request.

Platform & InfrastructureEXCLUDED
  • Fabric provisioning
  • Environment setup
  • Licensing activities
  • Infrastructure deployment
Migration ExecutionEXCLUDED
  • Physical migration of datasets
  • Moving data into Bronze/Silver/Gold
  • Production cutovers
  • Execution activities
DevelopmentEXCLUDED
  • Modifying SSIS packages
  • ETL redesign / redevelopment
  • New pipeline development
  • BI report build or redesign
Data RemediationEXCLUDED
  • Data quality fixes
  • Data cleansing
  • Master data remediation
  • We surface issues; Skyworks resolves
Operations & SupportEXCLUDED
  • Ongoing monitoring
  • Long-term administration
  • Post-engagement support
  • Platform access beyond term
Broader SystemsEXCLUDED
  • Deep ingestion of non-ODS systems
  • Independent upstream/downstream assessment
  • Reference systems = lineage context only

  • 03Make SMEs available for weekly 1-hour working sessions.
  • 04Supply business domain context and data-ownership information.
  • 05Participate in weekly status reviews and milestone sign-off gates.
