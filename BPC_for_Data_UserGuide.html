BPC for Data — how to use it

BPC for Data is the console that runs a data-estate assessment end to end: it connects the sources, catalogs and enriches them, maps lineage, and hands over the findings. This guide walks the flow and calls out what each page shows you, section by section.

1Sign in
Pick a profileOn the first screen choose Start Fresh Project to begin a new, blank assessment, a saved Client project, or the Mock Client (a fully populated example you can explore).
PIN to openSaved client and mock projects are gated — enter 2345 to open. Start Fresh opens with no PIN.
Pick a roleEach role card shows a one-line description of the person. What you can open depends on the role (next section).
2Roles — who sees what
C-LevelOverview only. The executive read on progress and outcomes.
Data EngineerOverview, Infrastructure, Collection, Unification.
Delivery / PMOverview and every assessment layer. No Admin Settings.
Solution ArchitectOverview, Infrastructure, Unification, Presentation.
Data AnalystOverview, Collection, Unification, Presentation.
AdminEverything, plus Admin Settings and project setup.
Pages are trimmed per role. A locked or hidden layer means that role does not work on it — the sidebar only lists what is relevant to you.
3Overview — the delivery timeline

The landing page for every profile. It answers one question at a glance: are we on track?

What you'll see here
  • A single Gantt timeline — week columns across the top, phase bands, and one bar per deliverable spanning the weeks it runs.
  • Status colors — green delivered, accent in-progress, red delayed, grey planned.
  • The "Now" marker — a green line at the current project week, so progress reads against today.
  • Delivered bars are clickable — an Open ↗ link opens the deliverable artifact.
  • Delayed bars are clickable too — click one to pop up why it is delayed, right on the bar.
Before an engagement is scheduled the timeline is empty — it appears once weeks and phases are set (a saved project, the Mock Client, or after Run full project).
4Set up sources with the FS Data Genie

On a Start Fresh project the Genie opens automatically and the rest of the app stays blurred until setup is done — this is where every new project begins.

Walk the groupsThe Genie asks one source group at a time in I → C → U order. Tick the sources in scope (ERP, CRM, databases, cloud, models) and continue.
FinishOn finish, your picks are provisioned as connections and the blur lifts. Only the sources you chose appear under Infrastructure.
Skipping the Genie. Use Run full project in the top bar to populate everything at once — that counts as finishing setup and lifts the blur. Sign out is always available if you need to leave.
5Assessment layers — I C U

The work is organized into layers. Each layer has sub-parts with live mocked metrics. Non-Infrastructure layers stay Locked until every selected source is connected.

Infrastructure · connectivity
  • All connections split into Sources / Cloud / LLM, each with an X / Y connected callout.
  • Usage meters — records ingested, freshness, compute, storage, LLM tokens, inference calls. Meters only appear once a category has a connected source.
  • Per-source cards — open one to see the prefilled endpoint, auth mode, and a secret reference (vault path / key name), never a raw password. Hit Test connection for a green result with ping time.
Collection · discovery
  • Domain Intelligence — business domains and data ownership captured with SMEs.
  • Automated Data Catalog — tables, schemas, columns and sizes ingested automatically.
  • Data Enrichment — usage patterns, access tools and frequency from query-log analysis.
  • Transformation — pipeline parsing and ETL / database assessment (role-gated for engineers and analysts).
Unification · lineage
  • End-to-End Visualization — lineage from upstream sources through transformations to downstream consumption.
  • Manual Additions — analyst-added lineage where API or metadata access is limited.
  • Ownership Mapping — dataset ownership and reference-data resolution.
6Presentation — the face of the studyHandover

The P layer is the packaged, decision-maker view — written for a C-suite audience. It is where the engagement narrative and every deliverable live.

What you'll see here
  • Highlight metrics — the top numbers for the estate (volume, pipelines, sources, weeks).
  • The engagement — the background, the outcome, and who owns the deliverables.
  • Current alignment — a % toward target with milestones (delivered / in-progress / planned).
  • Usage meters — live usage & utilization gauges, shown here when an admin turns them on (see §8).
  • Deliverables grid — every output with its status and an Open button when delivered.
  • Classification, Impact & Backlog, Enablement — the sub-parts that package findings for the client to act and own.
7Run full project — see it end to end
The fastest way to understand BPC for Data. On a Start Fresh project the Run full project button (top bar) populates the entire engagement at once — so you can see how a finished project looks without doing every step by hand.
What it doesConnects every source, then presents the whole engagement as complete: all connections live, usage meters active, every deliverable delivered and openable, alignment at target, all milestones done, and the schedule (active weeks + phases) filled in — so the Overview timeline reads fully green.
Why use itIt's the demo path — a start-to-finish picture of a real engagement. Use it to walk stakeholders through the flow before any real sources exist.
ResetSigning out of a Start Fresh session discards the run and the next Start Fresh opens blank again.
Prefer a ready-made example? Open the Mock Client profile from the login screen — it's permanently populated end to end, no run needed.
8Save, manage & admin
Save as projectWhile in Start Fresh, use the Save as project bar to name and store the current session. It becomes a saved client project, carrying over the sources you set up.
Delete a projectOn the login screen use the trash icon on a client project and enter PIN 2345. Start Fresh cannot be deleted.
Admin Settings · admin only
  • Upload SOW — drop a Statement of Work PDF; engagement, deliverables and systems are extracted to build the project. No financial figures are ever read or shown.
  • Enter project details — a manual form (no SOW or JSON needed): client name, engagement, weeks, narrative, highlight metrics and alignment.
  • Add profile from JSON — the advanced path: upload a prepared client config.
  • Presentation access — a toggle that shows the live usage & utilization meters as tiles on the Presentation page for executive viewers.
  • Stored profiles — every project with its type chip (Start Fresh / Client / Mock Client) and a delete action.
Sign-out safety. Leaving a Start Fresh project with unsaved sources prompts you to save or discard first, so nothing is lost by accident — and the next fresh project always starts clean.
